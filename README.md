# TinyBERT для классификации DGA доменов

Этот проект представляет собой реализацию классификации DGA (Domain Generation Algorithm) доменов с использованием модели TinyBERT. Модель обучается различать между легитимными доменами (benign) и доменами, сгенерированными алгоритмами DGA.

## 🎯 Особенности

- **Эффективная модель**: Использует TinyBERT для быстрой и точной классификации
- **Высокая производительность**: Оптимизировано для работы с большими объемами данных (16млн+ примеров)
- **Детальная оценка**: Включает ROC-AUC, precision-recall кривые, анализ ошибок
- **Гибкий инференс**: Поддержка одиночных доменов и батчевой обработки
- **Визуализация**: Автоматическое создание графиков метрик и результатов

## 📋 Требования

Установите зависимости:

```bash
pip install -r requirements.txt
```

## 📊 Формат данных

Данные должны быть в формате JSONL, где каждая строка содержит:

```json
{"domain": "example.com", "threat": "benign"}
{"domain": "qgcyquqgygcsaausi", "threat": "dga"}
```

- `domain`: доменное имя для классификации
- `threat`: метка класса ("benign" или "dga")

## 🚀 Использование

### 1. Обучение модели

```bash
python train.py \
    --data_path /path/to/your/data.jsonl \
    --model_save_path ./trained_model \
    --batch_size 32 \
    --num_epochs 3 \
    --learning_rate 2e-5
```

#### Основные параметры обучения:

- `--data_path`: путь к файлу с данными в формате JSONL
- `--max_samples`: ограничение количества примеров (для тестирования)
- `--batch_size`: размер батча (по умолчанию: 32)
- `--num_epochs`: количество эпох (по умолчанию: 3)
- `--learning_rate`: скорость обучения (по умолчанию: 2e-5)
- `--test_size`: доля тестовых данных (по умолчанию: 0.2)
- `--val_size`: доля валидационных данных (по умолчанию: 0.1)
- `--output_dir`: директория для результатов обучения
- `--model_save_path`: путь для сохранения модели

### 2. Инференс

#### Классификация одного домена:

```bash
python inference.py \
    --model_path ./trained_model \
    --domain "suspicious-domain.com"
```

#### Классификация нескольких доменов:

```bash
python inference.py \
    --model_path ./trained_model \
    --domains "domain1.com" "domain2.com" "domain3.com"
```

#### Классификация из файла:

```bash
python inference.py \
    --model_path ./trained_model \
    --input_file domains.jsonl \
    --output_file results.json \
    --output_format json
```

#### Параметры инференса:

- `--model_path`: путь к обученной модели
- `--input_file`: файл с доменами
- `--domain`: один домен
- `--domains`: список доменов
- `--output_file`: файл для сохранения результатов
- `--output_format`: формат вывода (json, csv, console)
- `--confidence_threshold`: минимальный порог уверенности
- `--show_only_dga`: показать только DGA домены
- `--batch_size`: размер батча для обработки

### 3. Детальная оценка модели

```bash
python evaluate.py \
    --model_path ./trained_model \
    --test_data /path/to/test_data.jsonl \
    --output_dir ./evaluation_results
```

Этот скрипт создаст подробный отчет включающий:
- Основные метрики (accuracy, precision, recall, F1, ROC-AUC)
- ROC и Precision-Recall кривые
- Матрицу ошибок
- Анализ различных порогов принятия решения
- Детальный анализ ошибок модели
- Калибрационные графики

## 🔧 Устранение неполадок

### Проблемы с памятью

Если возникают проблемы с памятью при обучении:
- Уменьшите `batch_size`
- Уменьшите `max_length`
- Используйте `gradient_accumulation_steps`

### Медленное обучение

- Используйте GPU если доступен
- Увеличьте `num_workers` для DataLoader
- Рассмотрите использование FP16 (включено автоматически)

### Низкое качество модели

- Увеличьте количество эпох обучения
- Настройте learning rate
- Проверьте качество разметки данных
- Увеличьте размер обучающей выборки
